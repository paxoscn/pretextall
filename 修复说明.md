# 修复说明 - 空格宽度计算

## 🐛 问题描述

右侧的文字被图片遮住了一部分，说明留出的空白没有按照图片的宽度正确计算。

### 问题原因

之前使用的是粗略估算：
```javascript
const spaceWidth = fontSize * 0.5; // 或 0.25
const spaces = Math.floor(imageSpaceWidth / spaceWidth);
```

这种估算方法的问题：
1. 不同字体的空格宽度差异很大
2. 同一字体在不同大小下，空格宽度与字体大小的比例也不同
3. 浏览器渲染时的实际宽度与估算值有偏差

## ✅ 解决方案

使用 Canvas API 精确测量空格宽度：

### 1. 创建 Canvas 测量工具

```javascript
// 在 init() 中创建
this.canvas = document.createElement('canvas');
this.ctx = this.canvas.getContext('2d');
```

### 2. 精确测量空格宽度

```javascript
// 设置字体
this.ctx.font = font; // 例如: "16px Arial"

// 测量空格的实际宽度
const spaceWidth = this.ctx.measureText(' ').width;
```

### 3. 计算需要的空格数量

```javascript
// 计算需要填充的宽度
const leftTextWidth = leftLine.width; // pretext 返回的实际文字宽度
const totalImageSpace = layoutInfo.imageWidth + layoutInfo.gap * 2;
const gapWidth = totalImageSpace + (layoutInfo.leftWidth - leftTextWidth);

// 使用精确的空格宽度计算数量
const spaces = Math.max(1, Math.ceil(gapWidth / spaceWidth));
```

## 🔧 技术细节

### Canvas measureText API

```javascript
const metrics = ctx.measureText(text);
// metrics.width - 文字的实际渲染宽度（像素）
```

优点：
- ✅ 精确：使用浏览器的实际渲染引擎
- ✅ 准确：考虑了字体、大小、样式等所有因素
- ✅ 一致：与页面渲染的宽度完全一致

### 宽度计算逻辑

```
总宽度 = 左侧文字 + 图片空间 + 右侧文字

需要的空格宽度 = 图片空间 + (左侧可用宽度 - 左侧实际文字宽度)

空格数量 = ceil(需要的空格宽度 / 单个空格宽度)
```

### 示例

假设：
- 左侧可用宽度：200px
- 左侧实际文字宽度：180px
- 图片宽度：50px
- 间距：10px
- 空格实际宽度：4px

计算：
```
图片总空间 = 50 + 10*2 = 70px
剩余空间 = 200 - 180 = 20px
需要填充 = 70 + 20 = 90px
空格数量 = ceil(90 / 4) = 23个空格
```

## 📊 对比

### 之前（估算）

```javascript
// 粗略估算
const spaceWidth = fontSize * 0.5; // 假设 16px 字体，空格 = 8px
const spaces = Math.floor(70 / 8); // = 8个空格
// 实际宽度 = 8 * 4 = 32px （不够！）
```

### 现在（精确）

```javascript
// 精确测量
const spaceWidth = ctx.measureText(' ').width; // 实际 = 4px
const spaces = Math.ceil(90 / 4); // = 23个空格
// 实际宽度 = 23 * 4 = 92px （正确！）
```
