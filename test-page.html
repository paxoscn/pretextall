什么是文字互动动画？

文字互动动画是一个创新的 Chrome 浏览器插件，它让自定义的动画元素能够跟随鼠标移动， 并与网页上的文字产生物理互动效果。当动画靠近文字时，会被文字"推开"， 创造出独特而有趣的视觉体验。

这个项目的灵感来源于 Cheng Lou 的 pretext 库，该库提供了高效的文字布局和测量方案。 我们借鉴了其核心思想，实现了实时的文字位置检测和物理计算系统。

✨ 核心特性

支持自定义图片上传、实时参数调节、流畅的 60fps 动画、智能的文字检测、 物理引擎驱动的互动效果，以及跨设备的设置同步。

技术原理

🔍 文字检测

使用 TreeWalker API 遍历 DOM 树，通过 Range.getClientRects() 获取每个文本节点的精确位置，并缓存到 Map 数据结构中以提高性能。

⚡ 物理计算

实现了基于反平方衰减的排斥力模型。计算鼠标与每个文字之间的距离， 在影响半径内生成排斥力向量，多个文字的力进行向量叠加。

🎬 动画渲染

使用 requestAnimationFrame 实现 60fps 流畅动画， 通过 CSS transform 属性触发 GPU 硬件加速， 确保动画性能和视觉效果的完美平衡。

密集文字测试区域

在这个区域，文字排列得更加密集，可以更好地测试动画的互动效果。 当你移动鼠标穿过这些文字时，会发现动画被多个文字同时影响， 产生更加复杂和有趣的运动轨迹。这种效果在新闻网站、博客文章等 文字密集的页面上尤为明显。

物理引擎会计算动画与每个文字之间的距离，并根据距离生成相应的排斥力。 距离越近，排斥力越强；距离越远，排斥力越弱。当多个文字同时在影响范围内时， 它们的力会进行向量叠加，最终决定动画的运动方向和速度。

你可以通过调整插件设置中的"互动强度"和"影响半径"参数来改变这种互动效果。 较高的强度会让动画被推得更远，较大的半径会让更多的文字参与到互动中。 尝试不同的参数组合，找到最适合你的设置！

使用场景

日常浏览：在阅读文章、浏览新闻时，轻微的互动效果可以让浏览体验更加生动有趣， 同时不会过分分散注意力。

演示展示：在进行屏幕录制或演示时，明显的动画效果可以吸引观众的注意力， 帮助引导视线到重要内容上。

创意表达：通过上传个性化的图片（如你喜欢的卡通形象、宠物照片等）， 可以让浏览器变成你独特的创意空间。

自定义建议

🎨 图片选择

推荐使用透明背景的 PNG 图片，尺寸在 100x100 到 500x500 像素之间。 可以选择表情符号、卡通形象、小动物、几何图形等。

⚙️ 参数调节

文字密集的页面建议增大半径、降低强度；文字稀疏的页面建议减小半径、提高强度。 根据个人喜好调整到最舒适的状态。

🎯 场景切换

为不同场景准备不同的配置：日常浏览用小尺寸低强度， 演示展示用大尺寸高强度，专注阅读用中等参数。

性能说明

插件经过精心优化，在大多数情况下都能保持流畅的 60fps 动画效果。 我们使用了多种性能优化技术：

  • 文字位置缓存，避免重复的 DOM 查询
  • 使用 Map 数据结构提高查找效率
  • GPU 硬件加速的 CSS transform 动画
  • requestAnimationFrame 同步屏幕刷新率
  • 被动事件监听器减少性能开销
  • 智能的 DOM 变化监听和更新策略

在极端情况下（如文字特别密集的页面），可能会有轻微的性能影响。 此时可以通过减小"影响半径"参数来降低计算量，或者暂时禁用插件。

常见问题

Q: 为什么某些网站上看不到效果？
A: 某些网站（如 Chrome 内部页面、扩展商店）出于安全原因不允许插件运行。 另外，使用 Shadow DOM 的网站可能无法完全检测到文字。

Q: 如何让效果更明显？
A: 增加"互动强度"和"影响半径"参数，或者使用更大、更醒目的图片。

Q: 感觉有点卡顿怎么办？
A: 减小"影响半径"可以降低计算量，提高性能。也可以使用更小的图片文件。

Q: 可以同时运行多个动画吗？
A: 当前版本只支持单个动画，多动画功能在未来版本的计划中。
