# 调试说明

## 问题：图片卡在页面右下角不动

### 已修复的问题

1. **移除了 CSS transition**
   - CSS 中的 `transition: transform 0.1s` 会干扰 JavaScript 的实时 transform 更新
   - 已从 `content.css` 中移除

2. **修正了动画元素的初始位置**
   - 添加了 `left: 0; top: 0;` 确保元素从左上角开始
   - 移除了可能导致问题的 `transition` 样式

3. **改进了 transform 计算**
   - 确保在计算 translate 时正确减去元素大小的一半
   - 添加了鼠标位置初始化检查

### 调试步骤

1. **打开浏览器控制台**（F12）
   - 查看是否有错误信息
   - 查看 "Text Interaction Animation initialized" 日志

2. **检查动画元素**
   - 在控制台运行：`document.getElementById('text-interaction-animation')`
   - 应该返回一个 div 元素

3. **检查鼠标位置**
   - 移动鼠标时，控制台应该显示 "Mouse position: x, y"
   - 确认鼠标位置被正确追踪

4. **检查 pretext 加载**
   - 在控制台运行：`window.pretext`
   - 应该返回一个包含 prepare、layout 等方法的对象

5. **检查动画循环**
   - 控制台应该显示 "Starting animation..."
   - 动画应该每帧更新

### 如果仍然不工作

1. **刷新页面**
   - 按 F5 刷新页面
   - 或者在扩展管理页面重新加载插件

2. **检查设置**
   - 点击插件图标
   - 确保"启用动画"开关已打开
   - 尝试调整参数

3. **使用测试页面**
   - 打开 `test-simple.html`
   - 这是一个简单的测试页面，文字较少

4. **查看元素样式**
   - 在控制台运行：
   ```javascript
   const el = document.getElementById('text-interaction-animation');
   console.log(el.style.transform);
   ```
   - 应该看到 transform 值在变化

### 常见问题

**Q: 动画元素不可见**
A: 检查 z-index 是否足够高（应该是 999999）

**Q: 动画不跟随鼠标**
A: 检查鼠标事件监听器是否正确添加

**Q: pretext 未加载**
A: 等待 500ms 后再初始化，或者检查 pretext-lib 文件夹是否存在

**Q: 动画卡顿**
A: 减小"影响半径"参数，降低计算量

### 手动测试代码

在控制台运行以下代码测试动画：

```javascript
// 测试动画元素是否存在
const el = document.getElementById('text-interaction-animation');
console.log('Element:', el);

// 手动设置位置
if (el) {
  el.style.transform = 'translate(100px, 100px)';
  console.log('Manually moved to 100, 100');
}

// 测试鼠标位置
document.addEventListener('mousemove', (e) => {
  console.log('Mouse:', e.clientX, e.clientY);
}, { once: true });
```

### 预期行为

- 动画元素应该跟随鼠标移动
- 当鼠标靠近文字时，动画应该被"推开"
- 动画应该流畅，没有卡顿
- 动画大小应该根据设置变化

### 修改后需要做的

1. 在 `chrome://extensions/` 页面点击刷新按钮
2. 刷新正在测试的网页（F5）
3. 移动鼠标测试效果
