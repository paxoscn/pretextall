# 测试步骤

## 🔧 准备工作

1. 在 `chrome://extensions/` 刷新插件
2. 打开测试页面（推荐 `test-wrap.html`）

## ✅ 测试清单

### 1. 基础功能测试

- [ ] 移动鼠标到段落上，图片跟随鼠标移动
- [ ] 图片显示正常（默认彩色圆圈或自定义图片）

### 2. 文字环绕测试

- [ ] 移动鼠标到段落**中间位置**
- [ ] 文字在图片左侧显示
- [ ] 文字在图片右侧显示
- [ ] 左右两侧文字之间有适当间距（图片占据的空间）

### 3. 文字恢复测试

- [ ] 将鼠标移动到段落上（文字重排）
- [ ] 将鼠标移出段落
- [ ] **文字恢复原始布局**
- [ ] 文字内容完全一致
- [ ] 样式保持不变

### 4. 样式保留测试

打开 `test-styles.html`：

- [ ] 紫色文字 - 颜色保持
- [ ] 渐变背景 - 背景保持
- [ ] 斜体字体 - 字体样式保持
- [ ] 边框文字 - 边框保持
- [ ] 所有其他样式保持不变

### 5. 性能测试

- [ ] 快速移动鼠标，动画流畅
- [ ] 没有明显卡顿
- [ ] 文字重排响应及时

### 6. 边界情况测试

- [ ] 鼠标在段落左边缘 - 文字在右侧
- [ ] 鼠标在段落右边缘 - 文字在左侧
- [ ] 鼠标在段落上边缘 - 文字在下方
- [ ] 鼠标在段落下边缘 - 文字在上方
- [ ] 鼠标完全移出 - 文字恢复原状

## 🐛 常见问题排查

### 问题 1：文字没有恢复原状

**检查：**
1. 打开控制台（F12）查看是否有错误
2. 确认 `isImageInElement` 返回 false
3. 检查 `element.dataset.modified` 是否被正确清除

**解决：**
- 已添加 `dataset.modified` 标记
- 改进了 `isImageInElement` 的碰撞检测逻辑
- 使用矩形重叠检测而不是点检测

### 问题 2：文字环绕效果不明显

**检查：**
- 图片大小是否太小（建议 60-100px）
- 鼠标是否在段落中间位置
- 段落文字是否足够长

**解决：**
- 调整"动画大小"到 80px
- 移动鼠标到段落正中间
- 使用 `test-wrap.html` 中的长段落测试

### 问题 3：性能问题

**检查：**
- 页面文字是否过多
- 是否有大量段落元素

**解决：**
- 减小"影响半径"
- 只在需要时启用插件
- 关闭其他占用资源的扩展

## 📊 预期结果

1. **鼠标移入段落**

2. **鼠标在段落内移动**
   - 文字实时调整
   - 始终环绕图片

3. **鼠标移出段落**
   - 文字立即恢复原状
   - 布局完全一致
   - 样式完全保留

### 性能指标

- 帧率：≥ 50 FPS
- 响应时间：< 50ms
- 内存使用：< 100MB

## 🔄 修复说明

### 本次修复内容

1. **改进碰撞检测**
   - 从点检测改为矩形重叠检测
   - 考虑图片的完整尺寸
   - 更准确地判断图片是否在元素内

2. **添加修改标记**
   - 使用 `dataset.modified` 标记已修改的元素
   - 确保恢复时能找到所有被修改的元素

3. **改进恢复逻辑**
   - 检查修改标记或文本内容
   - 恢复文本内容
   - 恢复 white-space 样式
   - 清除修改标记
